write once , run anywhere?

NO！很多安卓设备的h5能力糟糕的让人惊讶。就算是apple，目前也还是不够好:(

所以为了尽量保体验。我直接开发在ios5以上的系统。

设置phoneGap

目前为止，phoneGap的最新版本是2.2.0 。在2.1.0时候使用childbrowser会有bug（如果你需要联合登陆。我相信你会用到这个的。）

安装的教程点击这里。

禁止页面拖动。

苹果的浏览器在默认情况下可以拖动页面。使用UIwebView当然也不例外。

尽管我们可以使用touchmove来禁止页面被拖动。

document.addEventListener('touchmove',function(e){
    e.preventDefault();
});

但是这样很容造成一些其他的问题。

其实在phoneGap里可以设置：

在项目中找到Cordova.plist - > UIWebViewBounce。将值改成NO。

就是这么简单。

注意：在phoneGap 2.4.0 中 已经统一在 xml 中修改了。

找到config.xml 修改成：

<preference name="UIWebViewBounce" value="false" />

但是这个也会造成一些其他的问题。比如我使用：active来改变按钮的样式就实效了。

不过后来经过尝试，对于绑定了touch事件，比如touchend，就可以激活：active伪类。

但是我们可能还是需要使用click，因为使用click，用户可以取消误点击。而touch事件没有办法取消的。

后来我的解决方案是 绑定一个空的touchend，然后绑定 click。

代码如下：

_btn.addEventListener('touchend',function(){});
_btn.addEventListener('click',function(){
    //do somthing...
});

跨域设置

还是在刚才的Cordova.plist 在 ExternalHost 下面添加一个字段（默认是item0），值是 * (代表所有的域名。当然为了安全起见，你可以只写你自己的域名)

设置完之后，我们就可以使用ajax去请求。

我们的应用使用到了canvas，所以canvas跨域的安全性限制也可以得到解决。

这个要比浏览器客户端方便很多。

注意 在 phoneGap 2.4.0中已经默认添加 * 。如果需要修改

去config.xml 中修改。

尺寸

对于ipad来说，屏幕有高清屏和非高清屏。我们的程序是基于非高清屏去设置的。因为我们的程序的ui基本上是基于CSS去实现的。所以一般不会有太大的问题。

我们需要在index.html里做如下设置。

<meta name="viewport" content="user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, width=1024" />

如果你的是iphone

<meta name="viewport" content="user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, width=320" />

把meta加入到head中。

滑动

ios支持原生的滑动，效果会比iscroll好很多。关键是，它真的很简单：

div{
    overflow:auto;
    -webkit-overflow-scrolling: touch;
}

甚至我们可以使用js来调用

_div.addEventListener('scroll',function(e){
    console.log(e.target.scrollLeft);
    console.log(e.target.scrollTop);
    //your code here.
});
